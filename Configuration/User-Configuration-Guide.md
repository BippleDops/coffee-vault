## Part 4: Workflows and Usage Patterns

### Daily Workflow

**Morning Routine** (5-10 minutes):

1. **Open Obsidian Coffee Vault**

2. **Create Today's Daily Note**:
   - Use Calendar plugin or
   - Cmd/Ctrl + P ‚Üí "Open today's daily note"

3. **Review Yesterday** (2 min):
   - Check yesterday's coffee logs
   - Note any insights

4. **Plan Today** (1 min):
   - Which beans to brew?
   - Any experiments planned?

**Brewing Session** (5 minutes per brew):

1. **Start New Coffee Log**:
   - Cmd/Ctrl + T (Templater hotkey)
   - Select "Coffee Log" template

2. **Pre-fill Known Info**:
   - Template suggests recent beans/method
   - Adjust dose/water if needed
   - Note grind setting

3. **Brew Coffee** (while brewing):
   - Watch time
   - Observe bloom
   - Note any issues

4. **Taste and Rate**:
   - Initial sip: first impressions
   - Cool to drinking temp: full evaluation
   - Rate 1-5 (use 0.5 increments)

5. **Quick Notes**:
   - 1-2 flavor descriptors
   - "Better than yesterday" or "Too bitter"
   - What to adjust next

**Evening Review** (Optional, 5 min):

1. **Review Today's Logs**:
   - Best brew of the day?
   - What worked well?
   - What to improve tomorrow?

2. **Update Daily Note Summary**:
   - Total coffees: X
   - Average rating: X.X
   - Notes: [brief reflections]

**Time Investment**: 5 min per brew + 10 min daily overhead = ~25 min/day for 2-3 brews

## Part 5: Troubleshooting

### Common Issues

#### Issue: Templates Don't Work

**Symptoms**: Clicking template does nothing or shows raw code

**Diagnosis**:
1. Is Templater installed and enabled?
   - Settings ‚Üí Community Plugins ‚Üí Check "Templater"

2. Is template folder configured?
   - Settings ‚Üí Templater ‚Üí Template folder location
   - Should be set to "Templates"

3. Are template files in correct location?
   - File explorer ‚Üí Templates folder ‚Üí Should see .md files

**Solution**:
```
1. Settings ‚Üí Community Plugins ‚Üí Templater ‚Üí Enable
2. Settings ‚Üí Templater ‚Üí Template folder ‚Üí Set to "Templates"
3. Restart Obsidian
4. Test: Create new note, Cmd/Ctrl+T, select template
```

#### Issue: Datacore Queries Don't Display

**Symptoms**: Seeing raw query code instead of results

**Diagnosis**:
1. Check if Datacore is installed:
   - Settings ‚Üí Community Plugins ‚Üí "Datacore"

2. Check if query syntax is correct:
   - Should start with ```datacore
   - Should end with ```

3. Check if data exists:
   - Do you have coffee logs in Coffee Logs folder?

**Solution**:
```
1. Install Datacore plugin
2. Create test coffee log
3. Verify query with simple test:
   ```datacore
   list from "Coffee Logs"
   ```
4. Should show list of log files
```

#### Issue: No Suggestions in Templates

**Symptoms**: Template creates note but all fields empty

**Diagnosis**:
1. Do you have existing coffee logs?
   - Suggestions require history to work

2. Is Datacore working?
   - Suggestions use Datacore queries

3. Are properties correct in existing logs?
   - Check frontmatter of previous logs

**Solution**:
```
1. Create 3-5 manual coffee logs first
2. Ensure each has: type, date, beans, brew-method
3. Then use template - should see suggestions
4. If still blank, check console (Ctrl+Shift+I) for errors
```

#### Issue: Obsidian Feels Slow

**Symptoms**: Lag when opening files, typing, or switching notes

**Diagnosis**:
1. How many files in vault? (Settings ‚Üí About)
2. How many plugins enabled?
3. Is sync running constantly?

**Solution**:
```
1. Disable unused plugins
2. Settings ‚Üí Datacore ‚Üí Enable caching
3. Reduce number of open panes
4. Close graph view when not using
5. Consider upgrading computer if vault is very large
```

### Performance Optimization

#### Query Optimization

**Slow queries usually have these issues**:

1. **No limits**:
```javascript
// ‚ùå BAD - Could return thousands
const allLogs = dv.pages('"Coffee Logs"');

// ‚úÖ GOOD - Limited scope
const recentLogs = dv.pages('"Coffee Logs"').limit(50);
```

2. **Sorting before filtering**:
```javascript
// ‚ùå BAD - Sorts everything
dv.pages('"Coffee Logs"').sort(...).where(...)

// ‚úÖ GOOD - Filters first
dv.pages('"Coffee Logs"').where(...).sort(...)
```

3. **Multiple queries for same data**:
```javascript
// ‚ùå BAD - Queries 3 times
const avgV60 = dv.pages(...).where(method === "v60")...
const avgChemex = dv.pages(...).where(method === "chemex")...
const avgAero = dv.pages(...).where(method === "aeropress")...

// ‚úÖ GOOD - Query once
const allLogs = dv.pages(...).array();
const avgV60 = allLogs.filter(l => l.method === "v60")...
const avgChemex = allLogs.filter(l => l.method === "chemex")...
```

**Benefits**:

- Easier navigation
- Better Datacore performance
- Clearer mental model

#### Sync Optimization

**If using Obsidian Sync or similar**:
1. Exclude large files from sync:
   - Settings ‚Üí Sync ‚Üí Excluded folders
   - Add: `.obsidian/workspace` (frequently changes)

2. Selective sync on mobile:
   - Only sync essentials
   - Exclude Visualizations/ folder

3. Reduce conflict frequency:
   - Don't edit same note on multiple devices simultaneously
   - Wait for sync to complete before switching devices
